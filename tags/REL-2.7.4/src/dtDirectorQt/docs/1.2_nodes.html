Nodes

      Director nodes make up the primary structure of a Director script. Each node performs a specific function, chaining these nodes together will provide you with the functionality necessary to design your own scripted events.  
 
      A node can be placed into the script by dragging them from the palette browser (Figure 1.2.1) into the main canvas view.  
 


Figure 1.2.1: Palette Browser.  
 

      There are a number of different types of nodes that can be used, below is a brief description of each type and what they are used for.  
 

Event Nodes


Figure 1.2.2: Event Nodes.

      Event nodes are your entry points within the Director script (Figure 1.2.2 shows a few examples of what an event node can look like). Event nodes are usually triggered during gameplay by actions performed by the user. These actions can be anything such as pressing a button or walking into an area.  
 

Action Nodes


Figure 1.2.3: Action Nodes.

      Action nodes are the "guts" of your script. An action node can be used to satisfy a condition that must be met or to perform an operation during gameplay and can be chained together with other nodes to make up the bulk of your script. All actions must be triggered through chaining, either by an event node or another action.  
 

Value Nodes


Figure 1.2.4: Value Nodes.

      Value nodes act as storage containers for raw data. Each contains a specific data type such as an integer or floating point. By chaining these together with other nodes, their values can be used for the benefit of the script.  
 
      One special type of value node is the reference. Reference value nodes are used to reference an existing value found elsewhere in the script. At any time, interacting with the reference value is the same as interacting with the value it is referencing. To define a reference, the Reference value node's reference name must match the name field of the value to be referenced (see Figure 1.2.5 below) and the referenced value must be within the same macro or in one of its parents.  
 


Figure 1.2.5: Referencing a Value.

 
 

Macros


Figure 1.2.6: Macros.

      Macros are containers that contain their own set of nodes inside them. Much like a parent/child relationship in any hierarchy, Macros are the children of their parent scripts. From a parent, they look similar to any other node and can be manipulated as if they were. However, stepping inside a Macro is like jumping to another script entirely. Macro's are provided as a convenient way to organize your script.  
 
      Creating a macro can be done via right click menu within the editing canvas. All nodes currently selected when creating a macro will automatically be moved into that macro.  
 

Link Nodes


Figure 1.2.7: Link Nodes.

      Link nodes allow macros the ability to expose inputs, outputs, and value links within their parent scripts. Figure 1.2.7 shows a few macros with these connector links on them. These links allow you to connect chains from the parent script to the macro and back out again.
