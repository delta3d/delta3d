Chaining


Figure 1.3.1: A simple chain.  
 

      Chaining is when multiple nodes are "chained" together via their input and output links (the protruding tabs visible on the left and right border of each node). The flow of a chain always flows from left to right beginning from an event node. This means that nodes within a chain are activated one at a time in sequencial order from left to right as long as the chain continues. Chains may be stopped or re-directed depending on certain conditions met which are determined by the currently activated node. For example, if we have a node that compares the value of a bool (Figure 1.3.2 below), it may branch the chain in one of two possible directions depending on the value of the bool to be compared.  
 


Figure 1.3.2: A Conditional Chain.

 
 
      Value links (the tabs located on the top and bottom of each node) are used to connect a node's property to a Value node within the script. While every node property contains their own default value that you can define in that nodes property browser, it is sometimes beneficial to re-direct the data for that property to a value node located somewhere in the script. By doing this, the script is able to modify the data for that value whenever necessary.
